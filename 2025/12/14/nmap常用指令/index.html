扫描类型 命令 特点
SYN 扫描（默认） nmap -sS 192.168.1.1 快速、隐蔽，需 root 权限

UDP 扫描 nmap -sU 192.168.1.1 慢且不可靠，但可发现 DNS/SNMP 等服务

实战组合命令

当然可以！以下是更新后的表格，新增一列 “参数详解”，对每条命令中的关键参数进行逐项解释：

场景 命令 参数详解
快速侦察 nmap -F -T4 --open 192.168.1.1 -F：仅扫描 Nmap 默认服务文件（nmap-services）中标记为“最常用”的约 100 个端口，大幅缩短扫描时间。
-T4：使用“激进”时序模板，加快扫描速度（适合可靠网络）。
--open：仅显示状态为 open 或 **open
全面评估 nmap -sS -sU -T4 -A -v -p- --reason 192.168.1.1 -sS：TCP SYN 扫描（半开放），速度快且较隐蔽。
-sU：同时进行 UDP 端口扫描（如 DNS、SNMP、DHCP 等）。
-T4：激进速度。
-A：启用操作系统探测（-O）、版本探测（-sV）、默认脚本（-sC）和路由跟踪（--traceroute）。
-v：详细输出模式。
-p-：扫描全部 65535 个端口。
--reason：显示每个端口状态的判定依据（如 “syn-ack”、“reset” 等）。
隐蔽扫描 nmap -T2 -f -D RND:5 --max-parallelism 10 192.168.1.1 -T2：“礼貌”时序，降低发包速率，减少被 IDS 发现的概率。
-f：将 IP 数据包分片（通常分成 8 字节片段），绕过某些基于完整包检测的防火墙。
-D RND:5：生成 5 个随机诱饵 IP，与真实扫描源混在一起，混淆日志追踪（真实 IP 默认标记为 ME）。
--max-parallelism 10：限制同时探测的端口数不超过 10，进一步降低流量特征。
Web 服务枚举 nmap -p 80,443,8080 --script http-title,http-headers,ssl-enum-ciphers 192.168.1.1 -p 80,443,8080：只扫描常见 Web 端口。
--script http-title：获取网页 <title> 内容。
http-headers：提取 HTTP 响应头（如 Server、X-Powered-By）。
ssl-enum-ciphers：枚举目标支持的 SSL/TLS 协议版本和加密套件，用于评估加密强度。
内网资产发现 nmap -sn 192.168.1.0/24 -oG live_hosts.txt && awk '/Up/{print $2}' live_hosts.txt > targets.txt -sn：禁用端口扫描，仅执行主机发现（Ping 扫描）。
-oG：以“grep 友好”格式输出结果（每行包含状态如 Status: Up）。
awk '/Up/{print $2}'：从输出中提取状态为 “Up” 的 IP 地址（第2列）。
最终生成 targets.txt 供后续深度扫描使用。
漏洞扫描 nmap -sV --script vuln --script-args mincvss=7.0 192.168.1.1 -sV：探测服务版本，为漏洞匹配提供基础。
--script vuln：运行所有归类为 “vuln” 的 NSE 脚本（如检测 Heartbleed、Shellshock、MS17-010 等）。
--script-args mincvss=7.0：仅报告 CVSS 评分 ≥ 7.0 的高危/严重漏洞（需脚本支持该参数）。

✅ 使用建议：

  • 在渗透测试初期，优先使用 快速侦察 和 内网资产发现。
  • 对关键目标再执行 全面评估 或 漏洞扫描。
  • 隐蔽扫描 适用于高安全环境或红队演练，但速度极慢。

💡 提示：可通过 nmap --script-help vuln 查看当前系统支持哪些漏洞检测脚本及其参数。
--open：仅显示开放端口，减少噪音。

安全与合规建议

✅ 合法使用原则：

  1. 必须获得书面授权，禁止扫描非授权目标。
  2. 避免影响业务：使用 -T2 或 --max-rate 50 限制扫描强度。
  3. 记录操作：所有扫描应保留日志（-oA + 时间戳）。
  4. 遵守政策：了解目标组织的渗透测试政策（PTP）。
  5. 最小权限原则：仅扫描必要端口和服务。

🛡️ 防御建议（给系统管理员）：

  • 配置防火墙丢弃异常包（NULL/Xmas/FIN-only）：
  • 启用 IDS（如 Suricata、Snort）检测 Nmap 扫描行为。
  • 关闭不必要的服务，减少攻击面。

总结备忘单

目的 推荐命令
主机发现 nmap -sn 192.168.1.0/24
快速端口扫描 nmap -F -T4 192.168.1.1
全面信息收集 nmap -A -T4 192.168.1.1
全端口+服务识别 nmap -sV -p- -T4 192.168.1.1
漏洞检测 nmap --script vuln 192.168.1.1
隐蔽扫描 nmap -T2 -f -D RND:5 192.168.1.1
结果归档 nmap -oA scan_$(date +%Y%m%d) 192.168.1.1

附录：Nmap参数速查表

参数 说明
-sS TCP SYN扫描
-sT TCP Connect扫描
-sU UDP扫描
-sA ACK扫描
-sP Ping扫描(主机发现)
-sL 列举目标(不扫描)
-p 指定端口
-p- 扫描所有端口
-sV 服务版本探测
-O 操作系统识别
-A 全功能扫描(版本+OS+脚本)
-T<0-5> 扫描速度(0-慢,5-快)
-n 禁止DNS解析
-oN 保存为普通文本
-oX 保存为XML格式
--script 使用NSE脚本
