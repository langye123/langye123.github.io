git-dumper使用

发表于2025-12-18|更新于2025-12-19|工具
|字数总计:395|阅读时长:1分钟|阅读量:

🔍 一、git_dumper 到底是什么？

git-dumper（Python 包名为 git_dumper）是一个自动化工具，专门用于从
“泄露的 .git 目录”中恢复完整的 Git 仓库源代码。

👉 这说明 .git 泄露已确认，完全可以恢复源码！

✅ 正确操作步骤：

第 1 步：安装 git-dumper（只需一次）

它会安装一个命令行工具 git-dumper 和一个 Python 模块 git_dumper。

第 2 步：创建输出目录（比如叫 recovered）
第 3 步：运行恢复命令（核心！）

⚠️ 注意：

  • URL 必须以 /.git/ 结尾（带斜杠！）
  • recovered 是本地空文件夹（工具会自动填充）
第 4 步：等待完成（几秒到几分钟）

你会看到类似输出：

第 5 步：还原出可读源码

✅ 现在 recovered/ 目录里就是完整的网站源代码！

📂 三、恢复后你能看到什么？

  • 所有源文件（PHP、JS、HTML、配置等）
  • 提交历史（git log）
  • 分支信息
  • 可能包含：
    • 数据库密码（在 config.php 中）
    • API 密钥
    • 内网地址
    • 删除过的敏感文件（通过 git log -p 查看历史 diff）

✅ 总结：你现在要做的

步骤 命令
1. 安装工具 pip install git-dumper
2. 创建输出目录 mkdir recovered
3. 恢复仓库 python -m git_dumper http://node.hackhub.get-shell.com/.git/ recovered
4. 提取源码 cd recovered && git checkout .

完成后，打开 recovered/ 文件夹，你就拿到了对方的完整源代码 💥
