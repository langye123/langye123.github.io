BurpSuite常见的问题和思考

什么是BurpSuite

简单来说是相当于客户端和服务端之间的传递者，不同的是它可以双向的修改，删除。

问题1 安装的时候提到了CA证书，它是什么？

我知道他是用来让浏览器相信https加密的拦截。类似宝塔界面的控制界面，需要信任的CA证书，访问的时候才能不报红。

它的命名在我的计算机是BurpSuite.der，命名可以随便写，但是后缀必须是以.der结尾。

问题2 如果要在浏览器进行拦截用到了插件Proxy SwitchyOmega 3？

是的在里面进行代理端口设置，让他走8080方便监控

我电脑全局开了系统代理，BurpSuite又监控8080端口，那么这两个都是作用于浏览器两个不同的端口流量是怎么走的呢？

来来，我给你讲一件，其实他们逻辑关系是谷歌浏览器输入网址，进行访问，因为Proxy SwitchyOmega 3
走的端口是本地127.0.0.1:8080,这个是你设置的。从8080端口出去后，马上被BurpSuite监听到进去后，如果你不配置
上游代理商，它就本本分分往外发送，跟个大傻子一样，被拦截，访问不了外网。你配置上游服务商，给VrayN的监听端口后，被
VrayN监听进去，通过它再发出去。就可以进行外网访问了。注意VrayN是Sock不是http所以不要填错了，本来错了，现在改过来了

最上面的那一栏 → 只能填 HTTP/HTTPS 代理 您填 127.0.0.1:10808 → Burp 傻乎乎地用 HTTP 协议去跟一个纯 SOCKS5 端口握手 → 当然被拒绝 → 完全不生效！ 下面的“SOCKS proxy”那一栏 → 专门就是给 SOCKS5 用的 您把 127.0.0.1:10808 填进去 → Burp 正确地用 SOCKS5 协议跟 V2RayN 对话 → 完美直连 → YouTube 秒开 + 明文全抓！
