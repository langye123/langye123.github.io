解释：

  • dnf 是 CentOS 的包管理器。
  • wget 和 curl 用于下载文件。
  • unzip 用于解压 zip 压缩包。
  • vim 用于编辑配置文件。
  • -y 表示自动确认安装。

2️⃣ 下载并安装 V2Ray

解释：

  • curl -Ls 下载官方安装脚本，-L 跟随跳转，-s 静默模式。
  • <( ... ) 将下载的脚本作为临时文件传给 bash 执行。
  • 脚本会自动下载并安装最新版本的 V2Ray。

安装完成后会生成：

  • 主程序：/usr/local/bin/xray
  • 配置文件：/usr/local/etc/v2ray/config.json
  • systemd 服务：xray.service

配置Xray（隐蔽模式：VLESS + REALITY协议）

生成必要密钥（UUID和X25519密钥对）：

生成UUID（您的私人密钥）：xray uuid

生成X25519密钥对（用于REALITY加密）：xray x25519

编辑配置文件：/usr/local/etc/xray/config.json

保存后，限制权限（防别人偷看）：sudo chmod 600 /usr/local/etc/xray/config.json

这个不能这样
Xray 服务是用 nobody 用户 运行的（看您的 service 文件：User=nobody），而 nobody 用户没有权限读取 /usr/local/etc/xray/config.json

客户端配置

开启bbr加速

BBR（全称 Bottleneck Bandwidth and Round-trip propagation time）是Google开发的一种超级聪明的TCP拥塞控制算法。简单说，它就像给您的服务器装了个“智能油门”🚀：
老算法（比如cubic）一遇到丢包或延迟就猛踩刹车，速度掉得很惨，尤其在国际线路（翻墙场景）下特别明显。
BBR不一样，它会实时探测网络的实际带宽和延迟，然后尽量用满带宽，几乎不怕丢包！
结果就是：延迟更低、速度更快、更稳定，开启后速度直接翻2-5倍，尤其对您这种VLESS + Vision + Reality的配置，效果特别明显

  1. 添加BBR配置
    echo "net.core.default_qdisc=fq" | sudo tee -a /etc/sysctl.conf 系统配置文件（/etc/sysctl.conf）末尾添加一行：告诉系统用“fq”（Fair Queue）作为默认队列规则。

    echo "net.ipv4.tcp_congestion_control=bbr" | sudo tee -a /etc/sysctl.conf同一个配置文件末尾添加一行：直接告诉系统“以后所有TCP连接都用BBR算法”重启也没事status

  2. 立即生效
    sudo sysctl -p

  3. 再次检查（这一步您会看到魔法发生！）
    sysctl net.ipv4.tcp_congestion_control

  4. 重启一下xray sudo systemctl restart xray

启动和管理服务

启用并启动Xray：

sudo systemctl enable xray 开机自启动

sudo systemctl start xray

检查状态：sudo systemctl status xray如果绿灯（active），就成功了

重启或停止：

sudo systemctl restart xray // 重启

sudo systemctl stop xray // 停止
