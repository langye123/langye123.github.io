sqlmap从小白到实战

安装sqlmap

  1. https://github.com/sqlmapproject/sqlmap 从官网进行下载来安装zip，解压后，请注意必须要用python环境
    最好python3.0以上。
  2. 详细我就不介绍了 https://blog.csdn.net/liguangyao213/article/details/122655237 直接去看它

写在前面的话sql注入精力与产出要平衡。

  1. 据我所知现在的waf以及各主流框架云平台做的太好了。
  2. 可以会，不可大力投入。

常用指令

python sqlmap.py -h cmd转到该目录路径之后解释，先python,在启动。

🔑 基础语法

✅ 1. 基础检测（判断是否存在注入）

  • 作用：自动检测 URL 中的参数（如 id）是否存在 SQL 注入。
  • 说明：sqlmap 会尝试多种注入类型（布尔盲注、报错注入、时间盲注等）。

✅ 2. 携带 Cookie（绕过登录验证）

  • 作用：在请求中带上 Cookie，用于访问需要登录的页面（如 DVWA）。
  • 注意：Cookie 值要用双引号包裹，避免 shell 解析错误。

✅ 3. 自动化非交互模式（推荐！）

  • 作用：自动回答所有提示（默认选 Y），适合脚本化运行。
  • 避免卡在：“是否跳过其他 DBMS？”、“是否包含全部测试？”等交互问题。

✅ 4. 指定数据库类型（提速关键！）

  • 作用：只使用 MySQL 的 payload，跳过 Oracle/PostgreSQL/MSSQL 等测试。
  • 效果：速度提升 5~10 倍！

✅ 5. 获取数据库信息

获取当前数据库名：

列出指定数据库的表：

  • -D dvwa：指定数据库名为 dvwa

导出表数据（dump）：

  • -T users：指定表名
  • --dump：导出所有字段和数据（常用于获取账号密码）

✅ 6. 提高测试深度（发现更多漏洞）

参数 默认值 说明
--level 1 1~5，越高测试越多参数（URL、Cookie、User-Agent、Referer 等）
--risk 1 1~3，越高越危险（3 会尝试 OR 1=1 可能导致逻辑错误）

💡 DVWA 推荐：--level=3 --risk=2

✅ 7. 处理 POST 请求

  • --data：指定 POST 数据，sqlmap 会测试其中的参数。

✅ 8. 使用代理（调试或防封）

  • 配合 Burp Suite 抓包分析请求。

✅ 9. 绕过简单 WAF（需谨慎）

  • --tamper：使用编码脚本混淆 payload。
  • 常见 tamper：
    • space2comment：空格 → /**/
    • randomcase：随机大小写（SeLeCt）
    • charencode：字符编码

⚠️ 注意：WAF 绕过需根据实际情况定制，盲目使用可能失效。

✅ 10. 保存结果 & 恢复会话

  • sqlmap 会自动在 output/ 目录缓存结果，下次直接使用已有数据。

用文件txt，保存请求运行sqlmap

python sqlmap.py -r “D:\SafeNetwork\sqlmap-master\项目\request.txt” –batch –level=5 –risk=3

比如您的请求里有两个参数：

app=search
q=我的

如果您想让 sqlmap 只在 q 参数上尝试注入，可以这样：-p 参数名

python sqlmap.py -r “D:\SafeNetwork\sqlmap-master\项目\request.txt” –batch –level=3 –risk=2 -p q

下方是终极sqlmap实战命令

sqlmap -r D:\SafeNetwork\sqlmap-master\项目\request.txt --batch --dbms mysql --technique=B -D mydb -T huaguoshan -C id --dump --flush-session --tamper="space2comment,randomcase"
