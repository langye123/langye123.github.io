ADB (Android Debug Bridge)

ADB 是 Android 开发与调试的“瑞士军刀”。无论是应用安装、日志抓取、屏幕录制，还是系统级调试，掌握 ADB 都能极大提升效率。

一、环境搭建：从零开始配置 ADB

1. 下载与解压

  • 官方来源：推荐从 Android SDK Platform Tools 下载最新版。
  • 解压：将下载的压缩包解压到任意目录（建议路径无中文和空格），例如：
    • Windows: D:\adb\platform-tools

2. 配置环境变量 (关键步骤)

为了在终端任意位置使用 adb 命令，必须将其添加到系统环境变量 Path 中。

  • Windows:
    1. 右键“此电脑” -> “属性” -> “高级系统设置”。
    2. 点击“环境变量”。
    3. 在“系统变量”中找到 Path，点击“编辑”。
    4. 新建一行，填入 ADB 的绝对路径（如 D:\adb\platform-tools）。
    5. 连续点击“确定”保存。

若输出版本号（如 Android Debug Bridge version 1.0.41），则安装成功。

3. 无线连接 (Wi-Fi 调试)

无需数据线，通过 Wi-Fi 调试（需先通过 USB 连接一次以开启端口）。

1. Logcat 日志抓取 (核心中的核心)

  • 日志级别：V (Verbose), D (Debug), I (Info), W (Warn), E (Error), F (Fatal), S (Silent)。

  • dumpsys: 获取系统服务详细信息 (非常强大)

2. 授权问题 (Unauthorized)

如果 adb devices 显示 unauthorized：

  1. 检查手机屏幕是否弹出“允许 USB 调试”的对话框，点击“始终允许”并确定。
  2. 如果是无线连接，确保之前通过 USB 授权过。
  3. 尝试撤销授权：在手机开发者选项中点击“撤销 USB 调试授权”，然后重新插拔。

5. 配合 Scrcpy 使用

虽然 ADB 功能强大，但如果需要图形化控制，推荐配合开源工具 Scrcpy。它底层依赖 ADB，但能提供低延迟、高画质的手机投屏与控制体验，且无需在手机上安装任何 App。

七、总结速查表

功能分类 常用命令简述
连接 adb devices, adb connect IP:5555, adb kill-server
安装/卸载 adb install -r apk, adb uninstall pkg_name, pm clear pkg_name
文件传输 adb push src dst, adb pull src dst
屏幕操作 shell screencap, shell screenrecord, shell input tap/swipe
日志分析 adb logcat, adb logcat -c, adb logcat > log.txt
系统信息 shell dumpsys, shell top, shell wm density/size
按键模拟 input keyevent 3(Home), 4(Back), 26(Power)

提示：所有命令均可在 adb shell 内部直接运行（去掉前缀 adb），也可在外部直接加前缀运行。根据场景选择最便捷的方式。
