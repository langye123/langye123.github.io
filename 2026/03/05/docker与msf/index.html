Centos Stream 9/10 安装docker

检查系统版本

cat /etc/redhat-release

移除旧版/冲突包

sudo dnf remove -y docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine docker-ce docker-ce-cli containerd.io

安装必须的工具

sudo dnf install -y dnf-plugins-core device-mapper-persistent-data lvm2

添加官方 Docker CE 仓库

安装最新 Docker Engine + 插件

sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

启动 sudo systemctl start docker

开机自启动 sudo systemctl enable docker

确认版本
docker --version

查看 Docker Compose 版本

docker compose version

用frp进行反向代理

cd /var/www/

wget https://github.com/fatedier/frp/releases/download/v0.67.0/frp_0.67.0_linux_amd64.tar.gz

tar -zxvf frp_0.67.0_linux_amd64.tar.gz

cd frp_0.67.0_linux_amd64

服务器的配置 vi frps.toml

cd /home/kali/Desktop/tools/frp_0.67.0_linux_amd64

客户端kali的配置 vi frpc.toml

服务器移动到
cd /var/www/frp_0.67.0_linux_amd64

服务器启动
./frps -c ./frps.toml

让程序在后台永久运行并记录日志
nohup ./frps -c ./frps.toml > frps.log 2>&1 &

客户端启动
./frpc -c ./frpc.toml

让程序在后台永久运行并记录日志

nohup ./frpc -c ./frpc.toml > frpc.log 2>&1 &

进入后端自动运行的时候，可以通过下面的命令找到pid,kill

ps -ef | grep frpc/s

理由：输出更清晰，直接看到 PID 和 PPID。如果你想彻底清理，可以通过 PPID 找到它的父进程一起杀掉。而且它能完整显示启动参数，确认你跑的是不是正确的配置文件。
场景二：我觉得 frpc 有点卡，或者想看看它占不占内存

ps aux | grep frpc/s

理由：你能看到 %CPU 和 %MEM。如果 %CPU 很高，说明它在疯狂重试连接或有死循环；如果 RSS 内存很大，可能有内存泄漏。
