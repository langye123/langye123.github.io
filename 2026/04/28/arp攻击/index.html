环境准备

在发起攻击前，必须确保网络环境配置正确，并精准锁定目标。

1. 虚拟机网络配置

为了让 Kali 能够像局域网内的真实主机一样与目标通信，VMware/VirtualBox 的网络设置至关重要。

  • 模式选择：选择 **桥接模式 (Bridged)**。
  • 关键设置：务必勾选 **“复制物理网络连接状态” (Replicate physical network connection state)**。
    • 作用：确保当你宿主机切换网络（如 WiFi 切有线）或断网时，虚拟机的虚拟网卡状态能同步变化，保持连接稳定性。

2. 查看本机信息

首先确认 Kali 自身的 IP 地址和网卡名称（如 eth0）。

  • 关注点：找到 inet 后的 IP 地址（例如 192.168.31.x）和网卡接口名。

3. 存活主机扫描 (快速)

使用 fping 快速找出局域网内有哪些 IP 是“活”的。

  • 命令优化：
  • 参数详解：
    • -g：生成目标 IP 列表（支持网段或起止 IP）。
    • -a：仅显示存活（Alive）的主机。
    • 2>/dev/null：将错误输出（如 “Host is unreachable”）重定向到空设备，使输出结果更清爽，只显示有响应的 IP。

4. 获取 MAC 地址 (精准)

ARP 欺骗的核心是 MAC 地址，因此必须获取目标的 MAC。fping 无法显示 MAC，需使用 nmap。

  • 输出分析：寻找目标 IP 对应的 MAC Address 和厂商信息（如 Xiaomi Communications 或 VMware）。
  • 技巧：-sn 表示仅进行 Ping 扫描（主机发现），不进行端口扫描，速度更快。

⚔️ 第二阶段：实施 ARP 欺骗 (中间人攻击)

ARP 欺骗的原理是伪造 ARP 响应，告诉受害者“我是网关”，同时告诉网关“我是受害者”，从而让流量流经你的机器。

2. 双向欺骗 (双窗口操作)

窗口 角色 命令 作用
窗口 1 欺骗目标 sudo arpspoof -i eth0 -t 192.168.31.135 192.168.31.1 告诉目标主机：我是网关 (192.168.31.1)。
窗口 2 欺骗网关 sudo arpspoof -i eth0 -t 192.168.31.1 192.168.31.135 告诉网关：我是目标主机 (192.168.31.135)。
  • 注意：arpspoof 会持续运行，直到你按 Ctrl+C 停止。停止后，目标设备的 ARP 表通常会在几十秒后自动恢复。

流量监听与分析

1. HTTP 流量监听 (urlsnarf)

urlsnarf 是 dsniff 套件的一部分，专门用于从流量中提取 HTTP URL。

  • 监听范围：默认监听 TCP 端口 80 (HTTP)、8080 和 3128。
  • 输出示例：你会看到类似 192.168.31.135 - - "GET http://example.com/page HTTP/1.1" 的日志。
  • 局限性：现代网络大部分是 HTTPS，urlsnarf 只能看到域名或乱码，无法看到具体参数（如密码）。

2. 高级抓包 (Wireshark)

如果需要更深层的分析，配合 Wireshark 使用。

  • 在 Wireshark 中，你可以过滤 http 或 tcp.port == 80 来查看具体的请求包。

攻击结束后，记得在两个终端窗口中分别按下 Ctrl+C 停止 arpspoof，受害者的网络通常会在短时间内自动恢复正常。
