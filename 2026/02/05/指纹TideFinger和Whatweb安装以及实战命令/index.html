1. TideFinger（潮汐指纹）在 Kali 上的安装

TideFinger 有 Python 版（经典）和 Go 版（TideFinger_Go，速度更快），Python 版规则更全，Go 版适合批量。

Python3 版（最常用，规则最丰富）

注意 上面的python版本我已经放弃不用了原因kali自带的python不允许安装新的安装包会破环原来的版本导致某个工具不能用，这个可以解决
**

虽然我已经非常努力想要python,但是最后还是，环境版本的问题，他需要某一个包，但是高版本的python环境已经舍弃了

Go 版（TideFinger_Go，更快更稳，推荐批量用）

注意去他的官网看看，就知道go并没有放出源码，但是有它已经编译好的realse文件直接拿过来用

最强实战命令（主人直接复制用）：

命令参数 适用版本 功能说明 默认值 典型使用示例
-u URL Python / Go 指定单个目标 URL 进行指纹识别 无（必填） python TideFinger.py -u http://target.com``TideFinger -u https://admin.target.com
-uf FILE Go 从文件中批量读取 URL 列表进行扫描 无 TideFinger -uf urls.txt
-h CIDR Go 指定 IP 或网段（如 C 段）进行资产探测 无 TideFinger -h 192.168.1.0/24
-p PORTS Go 指定要探测的端口（可多端口） 80,443（部分实现） TideFinger -h 10.10.1.0/24 -p 8080,8443
-m NUM Python 设置指纹匹配的并发线程数 50 python TideFinger.py -u http://x.com -m 300
-t SECONDS Python 设置 HTTP 请求超时时间（秒） 5 python TideFinger.py -u http://slow.com -t 10
-pt SECONDS Go 设置每个目标的探测超时时间 约 10–30（依环境） TideFinger -uf targets.txt -pt 120
-p 1 Python 启用代理池扫描（需 proxys_ips.txt） 不启用 python TideFinger.py -u http://x.com -p 1
-nobr Go 禁用弱口令爆破功能 启用（默认） TideFinger -h 10.0.0.0/24 -nobr
-nopoc Go 禁用 POC 漏洞自动验证 启用（默认） TideFinger -u http://x.com -nopoc
-pd Go 显示有效 POC 的请求包与响应包详情 不显示 TideFinger -u http://vuln.com -pd
-np Go 跳过 ICMP Ping 探活，直接扫描端口 执行 ping（默认） TideFinger -h 192.168.1.1/24 -np
  • Python 版：轻量、适合单点快速识别，依赖 requests, lxml, bs4。
  • Go 版（TideFinger_Go）：高性能、支持大规模扫描、内置 23,000+ 指纹、含 POC 与爆破模块，推荐用于红队/企业安全测绘。
  • 代理文件要求（Python 版）：必须命名为 proxys_ips.txt，每行格式为 IP:PORT，如 124.225.223.101:80。
  • 批量扫描建议：Go 版原生支持 -uf，Python 版需配合 shell 循环实现。

whatweb

kali自带
识别网站使用的技术栈（CMS、JS 框架、服务器等）比较落伍了

手动更新插件
sudo apt update && sudo apt install --only-upgrade whatweb

若需 –log-json

sudo gem install json

✅ WhatWeb 常用参数与实战命令列表

序号 参数 说明 默认值 实战命令示例
1 <URL> 指定单个目标 URL（必填） 无 whatweb http://target.com
2 -v 详细模式（显示插件匹配过程、HTTP 头等） 关闭 whatweb -v https://admin.target.com
3 -a LEVEL 设置扫描强度等级：• 1 = 隐秘（1 次请求）• 3 = 激进（少量额外请求）• 4 = 深度（尝试所有插件） 1 whatweb -a 4 -v http://target.com
4 -i FILE 或 --input-file=FILE 从文件读取多个 URL（每行一个） 无 whatweb -i urls.txt
5 --log-json=FILE 将结果保存为 JSON 格式 不输出 whatweb -i urls.txt --log-json=results.json
6 --log-brief=FILE 将结果保存为简洁文本格式 不输出 whatweb -i urls.txt --log-brief=out.txt
7 --user-agent="UA" 或 -U "UA" 自定义 User-Agent 字符串 "WhatWeb/0.5.5" whatweb -U "Mozilla/5.0" http://target.com
8 --header="Key: Value" 或 -H "Key: Value" 添加自定义 HTTP 请求头 无 whatweb -H "Referer: https://google.com" http://target.com
9 --proxy=HOST:PORT 通过 HTTP/HTTPS 代理扫描 无 whatweb --proxy http://127.0.0.1:8080 http://target.com
10 --proxy=socks5://HOST:PORT 通过 SOCKS5 代理（如 Tor）扫描 无 whatweb --proxy socks5://127.0.0.1:9050 http://target.com
11 -t SECONDS 设置每个请求的超时时间（秒） 10 whatweb -t 15 http://slow-target.com
12 --no-errors 隐藏连接错误、超时等报错信息 显示错误 whatweb --no-errors -i targets.txt
13 -l 或 --list-plugins 列出所有可用插件（约 1800+） 无 whatweb -l
14 --info-plugins="keyword" 查看特定插件的详细信息 无 whatweb --info-plugins="wordpress"
15 --color=never 禁用彩色输出（适合日志记录） 启用颜色 whatweb --color=never http://target.com
16 --follow-redirect=WHEN 控制重定向行为：always（默认）、never、http-only 等 always whatweb --follow-redirect=never http://redirect-site.com
17 --max-redirects=NUM 设置最大重定向次数 10 whatweb --max-redirects=3 http://target.com
18 -p PORTS 指定端口（需配合 IP 使用） 80/443 whatweb 192.168.1.100 -p 8080

🔧 补充：组合命令实战场景

场景 完整命令
批量扫描 + JSON 输出 + 伪装浏览器 whatweb -i urls.txt --log-json=scan.json -U "Mozilla/5.0 (Windows NT 10.0)"
深度扫描 + 代理 + 超时 20 秒 whatweb -a 4 -v --proxy http://127.0.0.1:8080 -t 20 http://target.com
内网 Web 服务识别（IP + 端口） whatweb 10.10.1.50 -p 8080,8443
静默批量扫描（无错误干扰） whatweb --no-errors -i assets.txt --log-brief=clean.log

⚠️ 注意事项

  • 所有命令均在 Kali Linux 终端 直接运行（无需安装）。
  • 批量文件（如 urls.txt）需每行一个完整 URL，支持 # 注释。
  • 使用 -a 4 可能触发 WAF，仅限授权测试环境。
