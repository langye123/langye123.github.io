常见的命令参数

✅ httpx 常用命令行参数详解表

参数 全称 / 别名 作用说明 典型用法示例
-u --url 指定单个目标 URL 或主机名 httpx -u https://example.com
-l --list 从文件中读取多个目标（支持域名、IP、CIDR、URL 混合） httpx -l targets.txt
-silent — 静默模式：仅输出结果 URL，无额外信息 httpx -l subs.txt -silent
-status-code — 在输出中显示 HTTP 状态码 httpx -l subs.txt -status-code
-title — 提取并显示网页 <title> 标签内容 httpx -l subs.txt -title
-tech-detect — 启用技术栈识别（基于 Wappalyzer 规则） httpx -u site.com -tech-detect
-ip — 显示目标解析出的 IP 地址 httpx -u example.com -ip
-server — 显示 HTTP 响应头中的 Server 字段 httpx -u example.com -server
-json — 以 JSON 格式输出完整结果（含状态码、标题、IP、技术栈等） httpx -u example.com -json
-p --ports 指定要探测的端口列表（配合 CIDR 或主机使用） httpx -u 192.168.1.1 -p 80,443,8080
-follow-redirects -fr 自动跟随 HTTP 重定向（301/302） httpx -u short.url -follow-redirects
-location — 输出最终跳转后的 URL（需配合 -follow-redirects） httpx -u t.cn/xxx -fr -location
-H --header 添加自定义 HTTP 请求头（可多次使用） httpx -u site.com -H "X-Forwarded-For: 127.0.0.1"
-mc --match-code 仅输出匹配指定状态码的结果 httpx -l all.txt -mc 200,302
-fc --filter-code 过滤掉指定状态码的结果（排除） httpx -l all.txt -fc 404,500
-ml --match-length 仅输出响应体长度匹配的项 httpx -l urls.txt -ml 1234
-fl --filter-length 排除指定响应长度的结果 httpx -l urls.txt -fl 0,23
-threads -t 设置并发线程数（默认 50） httpx -l big.txt -threads 100
-timeout — 设置每个请求的超时时间（秒，默认 10） httpx -u slow.site -timeout 5
-rate-limit — 限制每秒请求数（避免触发风控） httpx -l targets.txt -rate-limit 30
--http2 — 强制使用或检测 HTTP/2 协议支持 httpx -u https://site.com --http2
-o --output 将结果保存到文件 httpx -l subs.txt -silent -o live.txt
-vhost — 启用虚拟主机（VHost）模糊测试（需配合字典） httpx -u example.com -vhost -w vhosts.txt
-path — 对每个目标追加指定路径进行探测 httpx -l ips.txt -path /admin,/login
-random-agent — 使用随机 User-Agent 发送请求 httpx -u site.com -random-agent
-no-color — 禁用彩色输出（适合日志记录） httpx -l subs.txt -no-color

以下是 最新版 httpx（Go 编译的 ProjectDiscovery HTTPX）实战命令清单，以 表格形式 呈现，包含 命令、用途说明、典型场景 三列，便于快速查阅与实战使用：

序号 命令（使用 httpx） 用途说明 典型应用场景
1 echo "example.com" | httpx -silent 静默探测单个目标是否存活 快速验证单个域名/主机是否提供 Web 服务
2 subfinder -d example.com -silent | httpx -silent -o live.txt 联动 Subfinder 枚举并筛选存活子域 子域名资产收集 → 存活过滤，构建初始攻击面
3 cat domains.txt | httpx -status-code -tech-detect -silent 输出状态码 + 技术栈指纹（如 Nginx、PHP） 初步识别目标技术架构，辅助漏洞匹配
4 httpx -u 192.168.1.0/24 -p 80,443,8080,8443 -silent 对 CIDR 网段指定端口进行 HTTP 探测 内网横向时快速发现 Web 管理界面或 API
5 httpx -u https://target.com --http2 -silent 检测目标是否支持 HTTP/2 协议 协议层面信息收集，辅助高级利用（如 HTTP/2 Smuggling）
6 httpx -l urls.txt -follow-redirects -location -silent 跟随重定向并输出最终跳转地址 解析短链接、登录跳转页、CDN 回源地址等
7 httpx -u target.com -H "User-Agent: Mozilla/5.0..." -silent 自定义请求头（如 User-Agent） 绕过基于 UA 的简单 WAF 或爬虫拦截
8 cat targets.txt | httpx -rate-limit 50 -threads 20 -silent 限速（50 req/s）+ 并发（20 线程） 大规模扫描时避免触发 IP 封禁或 DoS 防护
9 httpx -u example.com -json -silent 输出结构化 JSON 格式结果 供 Python/Shell 脚本解析，用于自动化流水线
10 cat all.txt | httpx -status-code -mc 200,302 -silent 仅保留状态码为 200 或 302 的结果 聚焦“有效页面”，排除 404/5xx 干扰项
11 httpx -u target.com -ip -title -server -silent 显示目标 IP、页面标题、Server 头 辅助判断是否使用 Cloudflare、WAF 或云厂商
12 httpx -l assets.txt -silent -o results.txt 从混合输入文件（域名/IP/CIDR/URL）批量探测 支持多种格式输入，适用于复杂资产列表
13 httpx -u slow-target.com -timeout 5 -silent 设置总超时为 5 秒 避免因个别慢响应拖慢整体扫描进度
14 subfinder -d example.com | httpx -silent | nuclei -t cves/ 与 Nuclei 联动实现“枚举→探活→漏洞扫描” 标准化渗透测试信息收集到漏洞验证流程
15 ln -s ~/go/bin/httpx ~/go/bin/httpx 创建软链接别名 httpx 避免与系统可能存在的 python3-httpx 冲突
