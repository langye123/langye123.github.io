BurpSuite抓包雷电模拟器包设置

发表于2026-02-27|更新于2026-03-04|安卓
|字数总计:941|阅读时长:3分钟|阅读量:

前景准备

我是基于原来学安卓逆向的前提下，准备对安卓进行抓包。简单网上找了找结合现在有的软件，BurpSuite+雷电模拟器。
模拟器+BurpSuite+抓包

思路

在BurpSuite代理监听器设置中，新添加一个监听的地址，添加绑定端口8081，绑定地址选择，指定地址输入自己局域网的IP4地址。
然后进入虚拟机的设置中，找到wlan设置，点进去选择手动填上代理地址，宿主机的ip4加上端口号。然后在BurpSuite中同样的位置选择导出CA证书
，(cacert.der更改为cacert.cer )将CA证书拖入虚拟机中，点击设置选择安全性和位置信息，点击 加密与凭据。点击信任的凭据CA证书，从SD卡中安装CA证书，或则我拖入进来之后直接长按，选择安装
注意这个时候安装是安装到用户信任的不是系统级信任证书。用Mt管理器，打开，
/data/misc/user/0/cacerts-added/ 把里面的CA证书移动到 /etc/security/cacerts/系统里面就行了。 CA证书名称是PortSwigger
在安装原来的顺序在点进去看看，就知道它进入了系统CA证书了

疑问

  1. 为什么用局域网ip4地址进行通信，我本地监听Google是用的是127.0.0.1?

    因为 127.0.0.1 是回环地址（Loopback），只代表当前设备自己。
    当 Burp 监听 127.0.0.1:8081 时，它只能被宿主机本机访问。
    而雷电模拟器本质上是一个 独立的虚拟设备，它有自己的网络栈和自己的 127.0.0.1。
    所以：
    模拟器中的 127.0.0.1 指的是模拟器自己
    127.0.0.1 指的是宿主机
    二者并不是同一个地址。 因此，要让模拟器访问宿主机上的 Burp，就必须使用宿主机的 局域网 IPv4 地址（例如 192.168.x.x），这样数据才能通过虚拟网络转发到宿主机。

  2. 还有就是雷电模拟器是用什么进行网络通信的？我进去看是wlan进行连接的？

    雷电模拟器底层基于 VirtualBox（虚拟化技术），默认采用 NAT 网络模式。
    其通信流程为：
    模拟器应用
    → 模拟器虚拟网卡
    → NAT 转换
    → 宿主机网卡
    → 外网
    模拟器中显示为 “WLAN”，只是 Android 系统层的网络类型标识。
    实际上并不是物理 WiFi，而是虚拟网卡模拟出来的无线网络接口。

  3. Vmos的kali它用了VMnet1,VMnet8,进行连接的，雷电模拟器好像没有？

    VMnet1、VMnet8 是 VMware 创建的虚拟网络接口：
    VMnet1：Host-Only 网络
    VMnet8：NAT 网络
    而雷电模拟器使用的是 VirtualBox 架构，并未像 VMware 一样在宿主机中显式创建 VMnet 网卡接口，而是封装在其内部虚拟网络驱动中。
    因此：
    VMware 会在宿主机看到 VMnet1 / VMnet8
    雷电模拟器则不会直接显示类似接口
    但原理上依然是 NAT + 虚拟网卡 + 宿主机转发
