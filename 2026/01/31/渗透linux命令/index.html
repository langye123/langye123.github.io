Linux 攻防命令速查与实战笔记

适用于红蓝对抗、CTF、应急响应、内网渗透等场景

🕵️‍♂️ 一、文件系统侦察（信息收集第一阶段）

1. ls —— 文件情报的第一眼

🔍 关注点：

  • 隐藏文件：.env（敏感配置）、.git（源码泄露）、.ssh（私钥）
  • 异常权限：如 777 或其他用户可写（可能被利用写入 WebShell）
  • 备份文件：config.php.bak、database.old 等，常含原始密码

2. tree —— 快速掌握目录结构

💡 用途：快速理解 Web 应用结构（如 /var/www/html），寻找上传点、日志路径、配置文件位置。

若未安装：apt install tree / yum install tree/dnf install tree 一般都没有安装

3. file —— 判断文件真实类型（比后缀可靠 100 倍！）

✅ 可识别：

  • ELF（Linux 可执行文件）
  • shell script / Python script
  • JPEG image data / gzip compressed data
  • ASCII text（可能是配置/密钥）

⚠️ 重要：攻击者常伪装 .jpg 后缀上传 WebShell，用 file 才能识破。

4. stat —— 文件元数据分析（时间线 & 篡改检测）

📊 输出包含：

  • Access（最后访问时间）
  • Modify（内容修改时间）
  • Change（元数据变更时间，如权限）

🔍 蓝队用途：判断文件是否被近期篡改（如 WebShell 上传时间 vs 日志时间）

📄 二、文本分析与日志处理（蓝队核心技能）

5. cat / tac / nl —— 查看文件的不同姿势

🎯 适用场景：快速检查配置、脚本逻辑、错误日志。

6. less —— 日志分析神器（交互式查看大文件）

⌨️ 快捷键：

  • /keyword → 搜索关键词
  • n / N → 下一个 / 上一个匹配
  • G → 跳到文件末尾（最新日志）
  • gg → 跳到开头
  • q → 退出

💡 比 cat 更适合 GB 级日志！

7. grep —— 文本检索之王（CTF & 应急必备）

📌 常用参数：

  • -i：忽略大小写
  • -r / -R：递归子目录
  • -v：反向匹配（排除）
  • -A 3 -B 2：显示匹配行前后几行（上下文）

8. awk —— 半自动日志分析（提取关键字段）

🧠 核心思想：按列处理结构化文本（日志、配置、CSV）

9. sed —— 批量替换 / 数据清洗

⚠️ 注意：-i 会原地修改文件，建议先备份！

10. wc —— 快速统计（判断规模）

⚔️ 三、权限、用户与提权（红队重点）

11. whoami / id / groups —— 当前身份确认

🔍 提权线索：若在 docker、lxd、adm 等特权组，可能有提权路径。

🔢 Linux 文件权限数字基础

字符 含义 数值
r 读 4
w 写 2
x 执行 1
- 无权限 0

💡 权限由三位八进制数组成，分别对应：
第一位：文件所有者（user）
第二位：所属组（group）
第三位：其他用户（others）

✅ 常用 chmod 权限组合速查表

权限 符号表示 含义 典型用途
600 -rw------- 仅所有者可读写 🔒 私密文件：• SSH 私钥（id_rsa）• 数据库密码文件• 敏感配置文件（如 /etc/shadow）
644 -rw-r--r-- 所有者读写，组和其他人只读 📄 普通文件：• 网页文件（HTML/CSS/JS）• 日志文件• 公共配置文件（如 nginx.conf）
700 -rwx------ 仅所有者可读、写、执行 🗂️ 私有目录或脚本：• 用户家目录（如 ~/.ssh）• 个人敏感脚本
755 -rwxr-xr-x 所有者全权，组和其他人可读+执行 🚀 可执行程序 / 公共目录：• Shell 脚本• Web 根目录（/var/www/html）• 系统命令（如 /bin/ls）
777 -rwxrwxrwx 所有人拥有全部权限 ⚠️ 极度危险！仅用于临时调试，绝不用于生产环境
664 -rw-rw-r-- 所有者和组可读写，其他人只读 👥 团队协作文件：• 共享配置• 开发项目源码
770 -rwxrwx--- 所有者和组全权，其他人无权限 🔐 团队私有目录：• 内部代码仓库• 受限工具目录

14. 查找 SUID 文件（⭐高频提权入口）

🔍 常见危险 SUID 文件：

  • /usr/bin/find
  • /usr/bin/nmap
  • /bin/mount
  • 自定义 C 程序（如 backup）

💡 利用方式：通过这些程序执行任意命令（GTFOBins 查询）

15. 查找敏感配置文件

database.conf、wp-config.php、settings.py 等含密码文件。注意这个只找到文件，不找文件里面的内容

🖥️ 四、进程与系统分析（攻防通用）

16. ps —— 查看运行进程

🔍 关注：

  • root 运行的可疑进程（如 minerd 挖矿）
  • 非标准路径的进程（/tmp/.X11-unix/...）
  • 进程名伪装（如 kthreadd 实为后门）

17. top / htop —— 实时资源监控

🚨 异常指标：

  • CPU 占用 100% 的未知进程
  • 内存突增（可能内存马）

18. uname / lsb_release —— 获取系统信息（找 Exploit）

🎯 用途：匹配本地提权漏洞（如 Dirty COW、CVE-2021-4034）

🌐 五、网络相关命令（渗透测试核心）

19. ip / ifconfig —— 网络接口信息

20. ss / netstat —— 查看网络连接（发现后门监听）

🔍 关注：

  • 127.0.0.1:3306 → MySQL 本地监听（可 SSRF 利用）
  • 0.0.0.0:6666 → 可疑后门端口
  • ESTAB 连接 → 当前活跃会话

21. ping / traceroute —— 网络连通性探测

💡 内网渗透时判断可达性。

22. curl / wget —— 万能 HTTP 工具

🔥 用途：

  • 下载 payload
  • 测试 API
  • 触发 RCE
  • 内网探测（配合代理）
