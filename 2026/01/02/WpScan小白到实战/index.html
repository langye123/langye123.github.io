🛠️ WPScan 实战手册（Kali Linux 专用）

🔍 前景介绍

WPScan 是一款专为 WordPress 网站设计的开源安全漏洞扫描工具，主要用于识别 WordPress 核心、插件、主题中的已知漏洞，枚举用户、检测弱密码、发现敏感文件等。它由 Ruby 编写，是渗透测试人员、红队成员和网站管理员常用的“WordPress 专用黑盒扫描器”。

💡 提示：WPScan 3.8.28 仍官方支持 -e, -U, -P, -t指的是线程 等简写参数（见 wpscan -h）

✅ 一、确认安装（Kali 默认已预装）

若未安装：

💡 Kali 中的 wpscan 包自动处理 Ruby 依赖，开箱即用。

✅ 二、获取 API Token

免费账户每日 25 次高级扫描配额，可获取完整 CVE、EXP 链接、CVSS 评分。

  1. 访问：https://wpscan.com/profile
  2. 注册 → 获取 Token
  3. 使用时加 -t YOUR_TOKEN

⚠️ 无 Token 也能扫，但漏洞信息不完整（仅提示“可能存在漏洞”）

✅ 三、核心命令结构

✅ 四、实战命令分类详解（适配 v3.8.28）

🔹 1. 基础信息探测（第一步）

  • -e p,t,u：
    • p = popular plugins（常用插件）
    • t = popular themes（常用主题）
    • u = users (默认枚举 ID 1-10)
  • --plugins-detection aggressive：主动探测插件（默认 passive 可能漏报）

⚠️ 注意：aggressive 会发送大量请求，易触发 WAF。仅在 passive 模式无结果时使用。

✅ 最佳实践：先用 passive（默认）侦察，再决定是否 aggressive。

💡 若 -e u 被禁用，可手动探测：

✅ 此操作不消耗 API 配额。

🔹 3. 插件/主题漏洞深度扫描（重点！）

📌 参数说明（来自 wpscan -h）：

  • vp = Vulnerable plugins ✅ 只查漏洞库中已收录的插件
  • vt = Vulnerable themes
  • 默认 -e 值为 vp,vt,tt,cb,dbe,u,m，但 显式指定更可控

✅ 关键总结：

  • 只有 -e vp 或 -e vt 会触发漏洞查询 → 消耗 1 次 API 配额
  • 不要对每个目标都用 -e p,t + token，这是常见浪费行为！
  • 先免费侦察（-e u,p,t 无 token），再对高价值目标用 -e vp,vt + token

🔹 4. 密码暴力破解

🔧 参数详解：

  • -U：指定用户名或文件（每行一个）
  • -P：密码字典路径
  • --throttle N：每次请求间隔 N 毫秒（防封 IP）
  • --max-threads N：并发线程数（默认 5，建议 ≤3 避免触发防护）
  • 攻击方式默认为 wp-login，也支持 xmlrpc 或 xmlrpc-multicall

✅ 爆破不消耗 API 配额，无需加 -t

⚠️ 法律提醒：未经授权的爆破属于违法行为！

🔹 5. 绕过基础防护（WAF / 限速）

参数说明：

  • --random-user-agent：每次请求随机 User-Agent
  • --proxy：流量走代理（配合 Burp Suite 分析）
  • --throttle：降低请求频率，避免被封
  • --disable-tls-checks：忽略 SSL 证书错误（测试环境常用）

--timeout：单次请求超时（秒） 在此版本已经被移除

🔹 6. 生成专业报告

✅ 若报告需包含 CVE 信息，则必须加 -t；否则可不加。

✅ 五、完整参数速查表（Kali + WPScan 3.8.28）

简写 全称 作用 示例
-u / --url — 目标地址（必填） -u https://wp.test
-e --enumerate 枚举内容 -e vp,vt,u
-U --usernames 指定用户名 -U admin
-P --passwords 密码字典 -P /wordlist.txt
没有简写 --api-token 启用完整漏洞库 -t abc123...
--random-user-agent — 随机 UA —
--proxy — 设置 HTTP 代理 --proxy http://127.0.0.1:8080
--throttle — 请求间隔（毫秒） --throttle 2000
-t / --max-threads — 并发线程数 --max-threads 3
-o --output 输出文件 -o result.json
-f --format 输出格式 json, cli, cli-no-color
--stealthy — 隐蔽模式（= passive + random UA） —
--plugins-detection — 插件检测模式 passive（默认）或 aggressive

✅ 所有简写均经 wpscan -h 验证，在 v3.8.28 中有效。

✅ 六、典型实战流程（红队视角）

✅ 配额节省黄金法则：

  1. 侦察阶段不用 -t
  2. 只对高价值目标用 -e vp,vt -t
  3. 避免 -e p,t + -t（浪费配额且低效）

✅ 七、注意事项 & 最佳实践

  1. 不要直接扫描生产环境：先在测试站练习。
  2. 更新漏洞库（虽自动更新，可手动触发）：
  3. 避免被封 IP：
    • 使用 --throttle 2000~5000
    • 线程数 ≤3
    • 优先用 --stealthy
  4. 结合其他工具：
    • ffuf：爆破 /wp-config.php.bak、.git/ 等敏感路径
    • Burp Suite：拦截登录请求，分析认证机制
    • nuclei：补充通用漏洞（如备份文件、XSS），但不能替代 WPScan

✅ 终极总结

场景 推荐命令 是否消耗配额
初步侦察 wpscan --url TARGET -e u,p,t ❌ 否
漏洞深度审计 wpscan --url TARGET -e vp,vt --api-token TOKEN ✅ 是（1 次）
密码爆破 wpscan --url TARGET -U user -P pass.txt ❌ 否
生成专业报告 加 -t 仅当需 CVE 信息 按需

🎯 记住：WPScan 是 WordPress 渗透的“黄金标准”，但 API Token 是有限资源——用在刀刃上！
