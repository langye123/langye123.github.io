迁移自己博客，从github到自己的服务器

发表于2024-07-13|更新于2024-07-14
|字数总计:1.8k|阅读时长:6分钟|阅读量:

前提自己系统是centox，登录到自己的服务器上，创建一个新的用户，sudo adduser boke

  1. 在服务器上安装必要的软件：
    安装 Git 和 Nginx：
    sudo apt install git
    sudo apt install nginx
    注意在第二步的时候自己配置nginx完全不行，用的宝塔控制页面进行安装的，最后成功了
  2. 配置 Nginx：
    修改 Nginx 的默认配置文件（通常位于 /etc/nginx/sites-available/default），将博客文件放到 /var/www/html 目录下。
    注意：我的nginx在 www/server/nginx/conf
    打开命令配置sudo vi /www/server/nginx/conf/nginx.conf

重新加载 Nginx 配置：
sudo service nginx reload
3. 创建 Git 裸库：
在服务器上创建一个 Git 裸库，用于保存博客文件：
cd ~
git init –bare blog.git

  1. 配置 Git Hooks：
    使用 Git 的 post-receive 钩子自动将最新内容同步到网站根目录：
    vim ~/blog.git/hooks/post-receive
    在文件中添加以下内容：
    #!/bin/sh
    git –work-tree=/var/www/html –git-dir=~/blog.git checkout -f 保存并赋予执行权限：

chmod +x ~/blog.git/hooks/post-receive

注意刚开始我又重新设置了一个角色，导致上一个命令不能正确赋予权限，我又重新到管理员的账号，执行了这个命令，注意把用户命令也加上chmod +x /home/your_username/blog.git/hooks/post-receive
5. 配置本地 Hexo 部署：
打开 Hexo 博客根目录下的 _config.yml 文件，找到 deploy 部分并修改：
deploy:
type: git
repository: ssh://user@your_server_ip:~/blog.git
branch: master
执行以下命令将博客部署到服务器：
注意命令的用户名，和服务器地址要改。
hexo clean
hexo generate
hexo deploy
6. 设置免密码登录：
在本地生成 SSH 密钥对（如果还没有）：
ssh-keygen -t rsa
将公钥上传到服务器：
ssh-copy-id user@your_server_ip
检查用户名公钥
cat ~/.ssh/authorized_keys

注意上传成功之后要在放行一下ssh规则

之后就成功了注意隐藏

# cat /etc/passwd 检查现在linx中所有用户
sudo passwd boke 给boke设置密码
su - boke 切换用户
sudo yum install git 使用使用root用户或sudo命令安装
ps -ef | grep nginx通过查看进程列表来确认 Nginx 是否在运行
sudo visudo 系统级别的配置文件,可以用来添加权限
sudo chown -R boke:boke /var/www/blog 这个命令用于改变 /var/www/blog 和其所有子目录及其内容的所有者（owner）和组所有者（group）。-R 参数表示递归（recursive），所以这个操作会应用于整个目录树。boke:boke 是指将所有文件和子目录的所有权更改为 boke 用户和 boke 组
