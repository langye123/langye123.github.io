什么是websock

自我理解，是基于tcp的一种连接，是进行直播弹幕，聊天的记录一种方式
他是以帧的形式进行传播，发送一个特殊的请求头，连接成功之后，两者，客户端和服务端都可以进行断开，接下来官方解释

官方解释

WebSocket 是一种能够在单个 TCP 连接上进行全双工通信的协议，非常适合用于实时消息传递，比如聊天页面。

WebSocket 的好处

  1. 实时性: WebSocket 可以在客户端和服务器之间保持一个持续的连接，使得消息可以立即发送和接收，确保实时性。

  2. 低开销: WebSocket 在建立连接后，不需要像 HTTP 一样在每次请求时发送头信息，从而减少了数据传输的开销。

  3. 双向通信: WebSocket 支持双向通信，这意味着服务器可以主动推送消息到客户端，用户在聊天应用中可以立即收到信息。

  4. 状态保持: WebSocket 连接可以在用户与服务器之间保持状态，增强了用户体验。

注意配置springsecrity也要放开这种请求方式

即使你已经放开了其他请求方式，WebSocket 连接仍然需要单独处理，因为它的工作方式与普通的 HTTP 请求不同。以下是一些原因：

连接持久性：WebSocket 是一种持久连接，而不是像 HTTP 那样的短连接。Spring Security 需要明确知道哪些端点可以建立这种持久连接。
握手过程：WebSocket 连接在建立时会进行一个握手过程，这个过程需要通过 HTTP 协议进行。如果没有明确放行 WebSocket 端点，Spring Security 可能会拦截这个握手请求。
路径匹配：你可能已经放开了某些路径，但 WebSocket 端点的路径可能没有包含在这些放行路径中。明确放行 WebSocket 端点可以确保这些连接不会被拦截。
安全性：即使你已经认证了用户，明确放行 WebSocket 端点可以避免不必要的安全检查，从而提高性能和连接成功率。
通过在 Spring Security 配置中添加 .requestMatchers(“/ws/**”).permitAll()，你可以确保 WebSocket 连接的握手过程不会被拦截，从而顺利建立连接。
