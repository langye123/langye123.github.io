其他的基于firewall常见命令

基础操作

查看 firewalld 状态

sudo systemctl status firewalld

启动 firewalld

sudo systemctl start firewalld

停止 firewalld

sudo systemctl stop firewalld

设置开机自启

sudo systemctl enable firewalld

📌 常用 firewall-cmd 命令

  1. 查看信息
    查看防火墙状态

sudo firewall-cmd --state

查看当前区域

sudo firewall-cmd --get-active-zones

查看所有开放的端口

sudo firewall-cmd --list-ports

查看所有服务

sudo firewall-cmd --list-services

  1. 管理端口
    开放端口（临时）

sudo firewall-cmd --add-port=8080/tcp

开放端口（永久）

sudo firewall-cmd --permanent --add-port=8080/tcp

sudo firewall-cmd --reload

移除端口

sudo firewall-cmd --permanent --remove-port=8080/tcp

sudo firewall-cmd --reload

  1. 管理服务
    允许某个服务（如 http）

sudo firewall-cmd --permanent --add-service=http

sudo firewall-cmd --reload

移除服务

sudo firewall-cmd --permanent --remove-service=http

sudo firewall-cmd --reload

  1. 其他常用操作
    重新加载防火墙规则
    sudo firewall-cmd --reload

允许 ICMP（ping）请求

sudo firewall-cmd --permanent --add-icmp-block-inversion

sudo firewall-cmd --reload

下面是iptables service

注意这个可能会让你安装sudo yum install iptables-services 这个注意一点就是防火墙规则可能丢失

放开icmp 请求 sudo iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT

sudo iptables -L -v -n 查看防火墙规则

sudo service iptables save 保存防火墙规则

sudo service iptables restart 重启当前防火墙规则

命令

ps aux | grep "http.server"

ps aux 显示所有用户的所有进程的详细信息
grep "http.server" 过滤出包含 “http.server” 的行

输出中最后一行通常是 grep 命令本身（因为 grep 进程也包含 “http.server” 字符串），这不是真正的 HTTP 服务器

sudo ss -tlnp | grep <PID>

查看系统上是否有进程在监听 TCP 4444 端口，并显示是哪个程序占用的

常识服务器理解问题

下面的为自己想不通的理解

服务器的安全组策略，和防火墙的区别

简单来说，服务器安全策略，是在防火墙之上的，只有安全组策略放开了，才能到服务器上的某个端口。两则互补

  1. 安全组策略（Security Group Rules
    安全组是云服务提供商（如 AWS、阿里云、腾讯云等）提供的一种虚拟防火墙，用于控制进出云服务器实例的流量。它们通常在虚拟网络层面上工作，作为云平台服务的一部分。

云平台管理：安全组由云服务提供商管理，通常通过云平台的控制台、API 或 CLI 工具进行配置。
实例级别：安全组规则应用于特定的云服务器实例或实例组，而不是单个端口或服务。
多规则组合：一个实例可以关联多个安全组，每个安全组包含一组入站和出站规则。
状态无关：安全组通常不跟踪连接状态，即它们不维护状态表来跟踪连接是否是已建立的或相关的。
2. 防火墙策略（Firewall Rules）
防火墙策略是指在操作系统层面上，通过使用如 iptables、firewalld 或其他防火墙工具来控制进出服务器的流量。

操作系统管理：防火墙规则由服务器管理员在操作系统层面上配置和管理。
详细规则：可以设置非常详细的规则，包括特定端口、协议、源/目标 IP 地址和范围等。
状态相关：现代防火墙通常可以跟踪连接状态，允许相关的返回流量自动通过，即使原始规则只指定了单向流量。
直接应用：防火墙规则直接应用于服务器的网络接口，与云平台无关。

服务器被安装挖矿程序 xmrig

  1. 首先top命令查看当前资源情况，发现cpu192% ，再仔细看发现xmrig

  2. ls -l /proc/2284890/exe 查看该pid命令所在目录，对应进程的可执行文件路径

  3. ps -ef | grep 2284890 在进程列表中查找包含“2437901”的行，或查看其父进程、启动命令行

  4. kill -9 1680

    pkill -f ‘watchdog.sh’
    pkill -f ‘xmrig’
    pkill -f ‘main.sh’
    按完整命令行匹配（-f）并终止所有匹配的进程

查看服务是否为自启动

查看本机“已设为开机自启”的服务 systemctl list-unit-files --type=service --state=enabled

查看当前“正在运行”的所有服务 无论是否自启动 systemctl list-units --type=service --state=running

查单条服务是否开机自启 systemctl is-enabled xray

如果你不想让 Xray 开机启动 sudo systemctl disable xray
